# Apple-Inspired Taste Reference for Virtual UWB

This file is a visual taste reference only.

It should influence polish, restraint, spacing, typography, and minimal chrome.

It must not be copied literally.

Virtual UWB is a functional Android mobile app, not an Apple-style product landing website.

---

## What To Borrow

Borrow these qualities:

- quiet confidence
- restrained visual design
- excellent spacing
- strong typography hierarchy
- minimal visual noise
- clean surfaces
- premium feel
- consistent component language
- subtle hairline borders
- minimal shadows
- clear action styling
- calm light backgrounds
- precise use of color

The UI should feel premium and intentional.

---

## What Not To Copy

Do not copy:

- full-screen product marketing tiles
- photography-first layout
- ultra-low information density
- web landing page structure
- Apple global web nav
- product configurator layout
- single-accent-only rule
- “UI disappears completely” approach
- huge empty sections
- marketing CTA patterns as the main screen structure

Virtual UWB needs functional app screens with controls, technical data, map canvas, event cards, and forms.

---

## Color Inspiration

Apple-like restraint means:

- use fewer colors
- use color intentionally
- avoid loud decoration
- keep backgrounds calm
- make semantic states clear

But Virtual UWB still requires semantic colors:

- indigo/purple for primary brand
- blue for anchors
- red for restricted zones and danger
- green for safe zones
- orange for EXIT events and warnings
- gray for secondary technical data

Do not force Apple’s single-blue-accent rule onto this app.

---

## Surface Inspiration

Use:

- off-white page background
- white or tonal cards
- subtle hairline borders
- restrained elevation
- soft rounded corners
- clean separation through spacing and surface color

Avoid:

- heavy shadows
- thick borders
- noisy gradients
- overly decorative surfaces
- dense admin blocks

Cards should feel premium but not flashy.

---

## Typography Inspiration

Use:

- confident titles
- readable body text
- careful hierarchy
- slightly tight modern headings
- muted labels
- stronger values

Do not make every text element the same size.

Technical values such as Lat/Lon, distance, bearing, timestamps, and event types must remain readable.

---

## Spacing Inspiration

Use whitespace as a premium design tool.

But do not remove information.

The app should feel breathable, not empty.

Use spacing to group:

- header
- status cards
- main panel
- controls
- data cards
- lists

Avoid cramped layouts.

---

## Button Inspiration

Use simple, clear button styles:

- primary filled or tonal buttons
- secondary outlined or tonal buttons
- pill-like status indicators
- compact chips

Avoid:

- too many different button styles
- oversized marketing CTAs
- decorative buttons

---

## Card Inspiration

Cards should be:

- rounded
- calm
- readable
- consistent
- lightly separated from background

Use cards for:

- events
- devices
- geofences
- system status
- selected tag details
- warning states

Do not make cards look like old admin panels.

---

## Applying This To Virtual UWB

The correct adaptation is:

Apple-like restraint + Material3 mobile components + Virtual UWB technical density.

Good result:

- modern Android app
- premium technical tool
- clean map screen
- elegant event list
- readable forms
- strong warning state
- clear realtime status

Bad result:

- Apple product website
- empty marketing page
- removed controls
- removed coordinates
- no semantic safety colors
- generic landing page
